# 分析流程与输出格式

## 提取爆款机制（核心步骤）

搜索同类高赞视频的目的不是拿它们的内容，而是分析它们**为什么能火**。从每条爆款视频中提取抽象层面的规律，这些规律就是"爆款机制"。

### 从每条爆款视频中提取什么

逐条分析搜索到的爆款视频，对每条提取以下维度：

1. **开头钩子类型** — 它怎么抓注意力的？
   - 反问型："你知道为什么…吗？"
   - 悬念型："这就是我没跑的原因"
   - 共鸣型："你有没有过那种感觉…"
   - 金句型：一句话定调
   - 场景型：直接进入一个具体画面
2. **情绪节奏** — 情绪怎么分布的？
   - 前段是什么情绪（温暖/愤怒/好奇）
   - 转折点在哪里
   - 后段推到什么情绪
   - 是均匀分布还是逐步推高
3. **转折手法** — 怎么制造认知反转的？
   - 人物反差（表面 vs 真实）
   - 情境反转（以为A结果B）
   - 视角切换（从一边看 vs 另一边看）
4. **收尾方式** — 最后落在什么上面？
   - 金句收尾（一句话让人截图）
   - 行动号召（转发/评论/点赞）
   - 留白（不说满，让观众自己想）
   - 回扣开头（首尾呼应）
5. **点赞高的关键原因** — 一句话总结这条视频为什么能火

### 汇总爆款机制表

把所有爆款视频的机制汇总成一张表：

| 爆款视频   | 开头钩子类型 | 情绪节奏                   | 转折手法 | 收尾方式 | 火的关键原因                     |
| ---------- | ------------ | -------------------------- | -------- | -------- | -------------------------------- |
| @xxx 889赞 | 反问型       | 前段温暖→中段冲突→后段释然 | 人物反差 | 金句收尾 | 戳中了"被看见"的需求             |
| @xxx 225赞 | 悬念型       | 前段好奇→后段共鸣          | 情境反转 | 留白     | 提供了一个"终于有人说出来"的表达 |

### 筛选可用的爆款机制

从汇总表中找出**最适合原视频内容的机制**（不是最多赞的视频，而是机制最匹配原视频素材的），作为改写的依据。选 2-3 组不同的机制组合，每组对应一篇改写。

## 二创核心原则（必须遵守）

原视频已经是爆款，有被验证的"爆款基因"。其他爆款视频只提供方法论，不提供素材：

1. **小编第三人称视角** — 你是文案小编，不是视频当事人。所有改写必须以小编视角讲述别人的故事，使用第三人称（"她""他""这位姐妹"）或第二人称对观众喊话（"姐妹们""你们见过吗"）。禁止用第一人称"我"代入当事人角色。你是旁观者、讲述者、评论者，不是当事人
2. **保留爆款核心** — 原视频的核心观点、钩子、情绪转折必须保留
3. **保留原文钩子和主体** — 原视频的开头钩子和主体叙事段落必须完整保留，不得用模板结构替换
4. **只换外壳不换内核** — 可以换叙事视角、开头方式、句式节奏，但核心论点和素材不能丢
5. **素材只用原视频的** — 所有改写的素材（场景、论据、故事）都只来自原视频，禁止把其他视频的内容加进来
6. **口语化不是文章化** — 短视频文案是口语表达，不是公众号文章。禁止"铺垫→递进→升华"的叙事弧线
7. **字数硬性要求** — 每篇改写文案必须 500-800 字，低于 500 字不合格

## 三阶段递进生成流程（核心）

先学会原视频怎么讲话，再用爆款机制优化表达方式。不是拿其他视频的内容来拼。

### 阶段一：复刻原视频（复刻度 ~80%）

**目标：** 学会原视频的讲话方式，确认抓住了钩子、主体和情绪。

**做法：**

1. 逐句拆解原视频文案，标注每个部分的功能：
   - 钩子句（开头抓注意力的那句话）
   - 主体段落（核心场景、论据、素材）
   - 情绪转折点（认知反转/情绪升级的地方）
   - 收尾句（最后落在什么上面）
2. 保留以上所有部分的结构和顺序，只做小幅改写：
   - 可以换词但保持原句式节奏
   - 可以删减重复但不能删核心素材
   - 不能加原视频没有的新论点或新场景
3. 输出一篇复刻稿，确认与原视频的情绪密度和口语感一致

**检查点：** 复刻稿读出来，节奏和口语感是否跟原视频接近？钩子是否还在？主体素材是否完整？

### 阶段二：用爆款机制轻度改写（复刻度 ~50%）

**目标：** 把爆款机制表中提取的某一组规律，应用到原视频的素材上。

**做法：**

1. 从爆款机制表中选一组规律（比如"共鸣型开头 + 均匀情绪分布 + 留白收尾"）
2. 用原视频的钩子、主体、转折、收尾的素材，按这组规律的节奏重新组织
3. 只改"怎么讲"，不改"讲什么"

**硬性约束：**

- 原视频的钩子核心意思必须保留（可以换说法，不能换意思）
- 主体叙事段落的核心场景和素材不能丢
- 情绪转折点必须保留，位置可以微调
- 不能加入原视频没有的场景、论据、故事
- 仍然保持口语化，不能变成文章叙事

### 阶段三：用爆款机制深度改写（复刻度 ~20-30%）

**目标：** 用另一组不同的爆款机制组合，对原视频做更大幅度的表达改写，但素材仍然只来自原视频。

1. 从爆款机制表中选另一组规律
2. 根据这组规律的风格，从模板描述表中匹配最合适的 prompt 模板，读取模板
3. **用原视频的素材填充模板**，不是用模板替换原文内容
4. 模板只决定"怎么讲"，不决定"讲什么"
5. 生成后对照原文检查：素材是不是都来自原视频？有没有混入其他视频的内容？

**选择模板** — 根据爆款机制组合的风格，从 SKILL.md 模板描述表中路由到最匹配的模板文件并读取。不要一次读取所有模板，只读需要的那一个。

## Validator（红线 + 字数 + 结构审查）

**生成每篇文案后必须立即执行此步骤，不可跳过，不可批量延后。**

### 第一步：红线审查

读取 `references/content-redlines.md` 中的红线守则，逐条审查改写文案：

1. **逐项过检** — 对照红线守则 checklist 逐条检查
2. **标注问题** — 如果触碰红线，明确标注哪一条、哪个词/句有问题
3. **修改文案** — 直接替换违规表述，不是删除整篇
4. **确认通过** — 修改后再次确认所有检查项通过

审查重点：

- 涉及真实人物：检查是否有捏造事实、主观定性、人身攻击
- 涉及社会事件：检查是否有消费灾难、娱乐化处理
- 涉及地域/民族：检查用词规范
- 涉及国际内容：检查称谓和用词准确性

### 第二步：字数审查

统计每篇改写文案的纯文字字数（去掉标点和标记符号后的汉字+数字+字母）：

- **合格范围：** 450–800 字
- **低于 450 字：** 不合格，必须扩写（补充原视频中的细节、增加小编点评、丰富场景描写），不可用凑字数的方式灌水
- **高于 800 字：** 不合格，必须精简（删减重复表述、合并相近句子），保留核心素材不动

扩写/精简后重新统计，直到字数落在 500–800 范围内。

### 第三步：结构审查

确认以下要素完整存在：

- [ ] 钩子句（开头抓注意力的那句话）
- [ ] 主体叙事段落（核心场景和素材完整）
- [ ] 情绪转折点（认知反转或情绪升级）
- [ ] 收尾句（落在明确的位置上）
- [ ] 小编第三人称视角（禁止第一人称代入当事人）
- [ ] 素材全部来自原视频（无其他视频内容混入）

### Validator 输出格式

每篇文案审查后追加：

```
**Validator：**
- 红线：✅ 通过 / ❌ 修改项列表
- 字数：XXX 字（✅ 合格 / ❌ 不合格，需扩写/精简至 450–800）
- 结构：钩子 ✅ 主体 ✅ 转折 ✅ 收尾 ✅ 视角 ✅ 素材 ✅
```
